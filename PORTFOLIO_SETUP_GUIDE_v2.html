Why Vercel is the cleanest option
  • One platform — frontend and contact API deploy together from the same repo in one step.
  • No sub-path — Vercel serves from root (/), no base: '/portfolio/' confusion.
  • No separate server — api/contact.js is auto-detected and served at /api/contact.
  • One secrets location — all env vars in the Vercel dashboard. No Cloudflare account, no wrangler.
  • SPA routing works out of the box — no 404.html hack needed.

Resend API

Never exposed to the browser

When a visitor submits the contact form, the browser POSTs to /api/contact. Vercel automatically routes that to api/contact.js. The function reads RESEND_API_KEY from Vercel's secure environment and calls the Resend API. No external server, no separate deployment.

Project structure

portfolio2-vercel/ ├── api/ │ └── contact.js ← Vercel serverless function (the contact API) ├── vercel.json ← build config + SPA rewrite rule ├── vite.config.js ← base: '/', dev proxy → port 3000 ├── .env.example ← server-side vars only, never in browser ├── package.json ← vercel CLI added as devDependency ├── src/ │ └── components/ │ └── Contact.jsx ← API_BASE = '' (same-origin, no env var) └── .github/workflows/ └── deploy.yml ← deploys to Vercel via CLI

02Set Up Resend

1

Create a Resend account

Go to resend.com and sign up for free.

2

Create an API key

Go to API Keys → Create API Key. Name: portfolio-contact, Permission: Sending access.

Copy the key — it starts with re_. You won't see it again.

⚠ This key goes into Vercel only. You will paste it into the Vercel dashboard in Part 3. It never goes in any file or GitHub secret.
3

Verify a sender domain Recommended

Go to Domains → Add Domain and follow the DNS steps for your provider. Until verified, use onboarding@resend.dev as FROM_EMAIL — Resend delivers to your Resend account email only in this mode, which is fine for confirming everything works.

03Deploy to Vercel

Option A — Vercel Dashboard (Easiest — no CLI needed)

1

Create a Vercel account

Go to vercel.com and click Continue with GitHub. This lets Vercel watch your repo and redeploy automatically on every push.

2

Import your repo

In the Vercel dashboard click Add New → Project → Import Git Repository. Select your portfolio repo.

3

Confirm build settings

Vercel auto-detects Vite. Confirm these match and click Deploy:

SettingValue
Framework PresetVite
Build Commandnpm run build
Output Directorydist

Your site and the contact API deploy together in ~30 seconds.

4

Set environment variables Required for contact form

After the first deploy, go to Project → Settings → Environment Variables. Set Environment to Production, Preview, Development for all of them:

KeyValueNotes
RESEND_API_KEY re_BccQuSUN_... Your full Resend key from Section 02
TO_EMAIL kuwalisean5@gmail.com Inbox that receives portfolio messages
FROM_EMAIL portfolio@yourdomain.com Verified Resend sender (or onboarding@resend.dev for testing)
ALLOWED_ORIGIN https://yoursite.vercel.app Your Vercel site URL — restricts which origins the function accepts
✓ These are server-side only. Vercel injects them into api/contact.js at runtime. They are never bundled into the browser build or visible in network requests.
5

Redeploy

Go to Deployments → latest deployment → ⋯ → Redeploy. This ensures the function picks up the new environment variables.

6

Test end-to-end

Open your Vercel URL, fill in the contact form, submit. You should receive an email at TO_EMAIL within seconds. Check Project → Functions → contact → View Logs if something doesn't arrive.

04Environment Variables Reference

All variables are server-side only — read by api/contact.js at runtime via process.env.*. None are embedded in the browser bundle.

Local .env file (development only)
RESEND_API_KEY=re_BccQuSUN_GjwnAJNQjGnT9ex6x1diAAUZ TO_EMAIL=kuwalisean5@gmail.com FROM_EMAIL=portfolio@yourdomain.com ALLOWED_ORIGIN=https://yoursite.vercel.app
⚠ Never commit .env to git. It is already listed in .gitignore. The committed .env.example has the same keys with placeholder values — safe to commit because it holds no real secrets.

Where to set them in production

Vercel dashboard → Project → Settings → Environment Variables. Set once; Vercel injects them into every function invocation automatically. No rebuild needed after changing a value — just redeploy.

05GitHub Actions (Automated CI/CD)

The included .github/workflows/deploy.yml deploys to Vercel automatically on every push to main. If you used the Vercel dashboard import in Section 03, Vercel already does this automatically — this workflow gives you more control and visibility in the GitHub Actions tab.

1

Link the project locally (one-time)

npm install npx vercel link # Signs in to Vercel and links to your project. # Creates .vercel/project.json — do NOT commit this file.

Open .vercel/project.json and copy orgId and projectId.

2

Add GitHub secrets

Go to repo → Settings → Secrets and variables → Actions → New repository secret:

Secret nameValue
VERCEL_TOKENvercel.com → Account Settings → Tokens → Create
VERCEL_ORG_IDorgId from .vercel/project.json
VERCEL_PROJECT_IDprojectId from .vercel/project.json
ℹ RESEND_API_KEY and email vars are NOT set here. They go in the Vercel dashboard (Section 04) where they're available at function runtime.
3

Push to main

Any push to main triggers the workflow. Watch it in the Actions tab of your GitHub repo.

06Local Development

Use vercel dev to run both the frontend and the serverless function together locally — identical to the production environment.

Terminal — full stack (recommended)
cp .env.example .env # fill in RESEND_API_KEY, TO_EMAIL, FROM_EMAIL, ALLOWED_ORIGIN npm install npm run dev:full # runs: vercel dev (port 3000)

Open http://localhost:3000. The contact form POSTs to /api/contact and the local function handles it exactly as it would in production.

Alternative — Vite only (no function)
npm run dev # Vite only on port 5173 — contact form will not send

07Troubleshooting

Symptom Likely cause Fix
Contact form error on live site Env vars not set in Vercel dashboard Project → Settings → Environment Variables; redeploy
Function logs: "configuration incomplete" RESEND_API_KEY / TO_EMAIL / FROM_EMAIL missing Add the missing variable in Vercel dashboard; redeploy
Email not arriving Wrong TO_EMAIL or FROM_EMAIL not verified Check Resend dashboard → Logs
CORS error in browser console ALLOWED_ORIGIN doesn't match Vercel URL Update the variable in Vercel dashboard; redeploy
GitHub Actions deploy fails Missing VERCEL_TOKEN / ORG_ID / PROJECT_ID secrets Add all three in repo → Settings → Secrets → Actions
404 on /api/contact api/contact.js not found or build error Check Vercel deploy logs; confirm file is at api/contact.js in project root
White screen locally Running npm run dev instead of vercel dev Use npm run dev:full; or open http://localhost:3000 not 5173
